# Agent Workflow Notes

- 每次推进 TODO 时，默认优先打包完成多个相关待办（建议 >=2 项），避免只完成一个小点就停下。
- 完成后需同步更新文档：
  - `TODO.md` 仅保留 pending，并更新本次迭代后的任务状态与下一步重大版本规划。
  - `PROGRESS.md` 记录本次已完成能力（从 TODO 迁移已完成事项）。
  - `README.md` 保持 onboarding 入口定位，不混入路线图细节。
- 文档分工：PRD 记录高层需求，PROGRESS 记录已实现历史，TODO 只保留待办，README 用于新人 onboarding。
